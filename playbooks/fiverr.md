- Core deliverable only
- 1 revision
- No source code
- Price: ~$50–150

## Order Intake Process

1. **Read the order brief fully** before starting anything
2. **Send a clarification message within 2 hours** of order placement — even if everything is clear, just confirm scope
3. Use this template:

```
Hey [Name], thanks for the order! Just confirming the scope:

- [Restate what you understood]
- [Any specific tool/platform they mentioned]
- Expected delivery: [date]

Let me know if anything needs adjusting before I start. I'll have updates for you by [date].
```

4. **Log the order** in the income tracker
5. **Start within 24 hours** of scope confirmation

---

## Delivery Process

1. **Test everything twice** before sending — never deliver untested work
2. **Write a delivery message** that explains what was built and how to use it:

```
Hey [Name], your [project] is ready!

Here's what I built:
- [Feature 1]
- [Feature 2]

How to use it:
1. [Step 1]
2. [Step 2]

Files are attached / deployed at: [URL]

Let me know if you have any questions. If everything looks good, I'd really appreciate a review!
```

3. **Deliver 12–24 hours early** when possible — gives buffer for revision requests
4. **Never deliver late** — request an extension early if needed

---

## Revision Handling

- **1 revision request:** Handle immediately, same-day
- **Scope creep:** Politely redirect — "That sounds like a great addition! It's outside the current scope but I can quote you for it."
- **Unreasonable client:** Document everything, escalate to Fiverr resolution only as last resort

---

## Review Acquisition

- Ask for a review in every delivery message (see template above)
- Follow up once, 24–48 hours after delivery, if no review yet:

```
Hey [Name], hope everything is working well! If you have a moment, a review would mean a lot — it helps other buyers know what to expect. Thanks!
```

- **Never pressure** — one follow-up max
- Respond to every review, positive or negative, within 24 hours

---

## Pricing Rules

- **Never discount more than 20%** for repeat buyers
- **Never negotiate below your floor price** — it trains bad clients
- **Charge for rush orders:** +30–50% for delivery in <24h
- **Quote extras upfront:** Always mention potential add-ons before starting

## Do's and Don'ts

**Do:**
- Communicate proactively
- Deliver early
- Over-document your work
- Upsell naturally in delivery messages
- Respond within 2 hours during business hours

**Don't:**
- Start work before scope is confirmed
- Deliver untested code
- Accept scope creep without extra payment
- Ignore messages for more than 4 hours
- Use the same generic description for every gig
